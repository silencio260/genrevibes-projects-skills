## 📱 Android Adaptive Icon — Samsung "Safe Zone" Fix

> ⚠️ **CRITICAL**: `adaptive_icon_foreground_inset` in `flutter_launcher_icons` is NOT a real parameter. It is silently ignored. Do NOT use it.

Samsung, Pixel, and other manufacturers apply aggressive circular/squircle/rounded masks to adaptive icons. Android's spec says only the **central 66%** of the icon is guaranteed to be visible (the "safe zone"). If your logo fills the full square canvas, it WILL be clipped.

### ✅ The Working Solution (Proven)

**Step 1 — Generate a "Safe" Foreground Asset using macOS `sips`**

Use the macOS built-in `sips` tool to shrink the logo to 60% and pad with black:

```bash
# Run from the project root (ai_chatbot/)
sips -s format png -z 614 614 assets/icon/app_icon.png --out assets/icon/app_icon_safe.png
sips -s format png -p 1024 1024 --padColor 000000 assets/icon/app_icon_safe.png --out assets/icon/app_icon_safe.png
```

> This creates `assets/icon/app_icon_safe.png`: a 1024×1024 image where the logo is shrunk to 614px (60%) and surrounded by 205px of pure black on each side — safely inside the 66% safe zone.

**Step 2 — Update `pubspec.yaml`**

```yaml
flutter_launcher_icons:
  android: "launcher_icon"
  ios: true
  image_path: "assets/icon/app_icon.png"         # Full size for iOS (keep this!)
  adaptive_icon_background: "#000000"
  adaptive_icon_foreground: "assets/icon/app_icon_safe.png"  # Shrunk for Android
```

> ⚠️ `image_path` must remain the original full-size icon. If you point it to `app_icon_safe.png`, the iOS icon will appear tiny.

**Step 3 — Regenerate Icons**

```bash
flutter pub run flutter_launcher_icons:main
```

**Step 4 — Done!** No XML editing required.

### Creating the Transparent Splash Logo
The `app_logo_transparent.png` must be a true PNG with alpha transparency (not a JPEG or a PNG with a black background). To regenerate it from the main `app_icon.png` on macOS, use a Swift script or similar tool to replace pure black (#000000) with 0% alpha.

## 🔔 OneSignal Notification Icons (Android)

When you change the app logo, you MUST also update the OneSignal notification icons. Android dictates that notification icons must be **pure white with an alpha transparency channel** (no colors, no solid backgrounds). If you use a colored app icon, it will show up as a solid white or gray square in the notification tray!

**Steps to Update:**
1. Create a simplified silhouette of your logo where the design is **pure white** (`#FFFFFF`) and the background is completely transparent.
2. Generate the required Android sizes (24x24, 36x36, 48x48, 72x72, 96x96).
   > **Tip:** Use the free [Android Asset Studio Notification Icon Generator](https://romannurik.github.io/AndroidAssetStudio/icons-notification.html) to automatically batch generate all the sizes from your transparent PNG.
3. Keep the file name as `ic_stat_onesignal_default.png`.
4. Replace the old icons in your Android source directories:
   - `android/app/src/main/res/drawable-mdpi/ic_stat_onesignal_default.png`
   - `android/app/src/main/res/drawable-hdpi/ic_stat_onesignal_default.png`
   - `android/app/src/main/res/drawable-xhdpi/ic_stat_onesignal_default.png`
   - `android/app/src/main/res/drawable-xxhdpi/ic_stat_onesignal_default.png`
   - `android/app/src/main/res/drawable-xxxhdpi/ic_stat_onesignal_default.png`

> **Note:** iOS automatically uses the main `AppIcon` generated by `flutter_launcher_icons`, so no separate OneSignal icon generation is needed for iOS.

## 🔄 When to Re-run This Process

If you ever change the app icon design, simply:
1. Replace `assets/icon/app_icon.png` with the new icon.
2. Re-run the `sips` commands above (Step 1) to regenerate `app_icon_safe.png`.
3. Re-run `flutter pub run flutter_launcher_icons:main` (Step 3).
4. Update your `app_logo_transparent.png`.
5. Generate the new `ic_stat_onesignal_default.png` notification icons to ensure your Push Notifications match your new branding.

## ❌ Things That DO NOT Work (Lessons Learned)

| Approach | Why It Fails |
|---|---|
| `adaptive_icon_foreground_inset: <N>` in pubspec.yaml | This key is **not supported** in `flutter_launcher_icons`. It is silently ignored. |
| Using `app_icon.png` (full black bg) as foreground | The solid black box fills the full canvas. The mask cuts through it. |
| Percentage insets (`17%`) in Android XML layer-list | AAPT does not support `%` units for `android:top/bottom/left/right` on `<item>` tags. |
| Using `app_logo_transparent.png` (transparent) as foreground | The transparent logo still fills 100% of the foreground canvas. The mask clips the edges. |
