# Firebase Infrastructure

## Overview

All GenRevibes apps use Firebase as the backend infrastructure. The starter kit handles Firebase Analytics, Crashlytics, and Remote Config. This skill covers the broader Firebase setup including Firestore, Cloud Storage, Cloud Functions (via Genkit), and deployment.

## Prerequisites

- Firebase project created in Firebase Console
- FlutterFire CLI installed: `dart pub global activate flutterfire_cli`
- Firebase CLI installed: `npm install -g firebase-tools`
- `firebase_core` dependency (included via starter kit)

## Architecture

```
project_root/
├── lib/
│   └── firebase_options.dart        # Generated by FlutterFire CLI
├── backend/
│   └── functions/
│       ├── src/
│       │   ├── flows/               # Genkit AI flows
│       │   │   ├── chat.flow.ts
│       │   │   └── image-generate.flow.ts

│       ├── index.ts                 # Cloud Functions entry point
│       ├── package.json
│       └── tsconfig.json
├── firebase.json                    # Firebase project config
└── .firebaserc                      # Firebase project aliases
```

## Setup Steps

### 1. Initialize Firebase in Flutter

```bash
flutterfire configure
```

This generates `firebase_options.dart` with platform-specific configs.

### 2. Initialize in main.dart

```dart
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);
  // ... then StarterKit.initialize()
}
```

### 3. Firestore Structure

Standard collection patterns used across projects:

```
firestore/
├── users/                    # User profiles (via starter kit database module)
│   └── {userId}/
│       ├── profile           # User profile data
│       ├── subscriptions     # Subscription status
│       └── usage             # Usage counters
├── chats/                    # Chat sessions
│   └── {sessionId}/
│       ├── metadata          # Session metadata
│       └── messages/         # Sub-collection of messages
│           └── {messageId}
├── images/                   # Generated images
│   └── {imageId}/
│       ├── metadata          # Generation params, prompt, style
│       └── variants/         # Image variations
└── app_config/               # App-wide configuration
    └── settings              # Global settings document
```

### 4. Cloud Functions (Genkit)

The backend uses **Firebase Genkit** for AI flows:

```typescript
// src/flows/chat.flow.ts
import { defineFlow } from '@genkit-ai/flow';

export const chatFlow = defineFlow({
  name: 'chat',
  // ... flow definition
});
```

**Deployment:**
```bash
cd backend/functions
npm install
npm run build
firebase deploy --only functions
```

### 5. Cloud Storage

Used for storing generated images, user uploads, etc.

```dart
// Storage bucket structure
gs://your-project.appspot.com/
├── images/
│   └── generated/
│       └── {userId}/
│           └── {imageId}.png
├── uploads/
│   └── {userId}/
│       └── {filename}
└── exports/
```

### 6. Security Rules

```javascript
// firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /chats/{sessionId} {
      allow read, write: if request.auth != null;
    }
  }
}
```

## Starter Kit Integration

The starter kit manages:
- **Firebase Analytics** → `StarterKit.analyticsBloc`
- **Firebase Crashlytics** → Automatic crash reporting
- **Firebase Remote Config** → `StarterKit.sl<RemoteConfigRepository>()`
- **Firebase Auth** → `StarterKit.sl<AuthRepository>()` (if provided)

Your app manages:
- **Firestore** → Feature-specific repositories
- **Cloud Storage** → Feature-specific data sources
- **Cloud Functions** → Called via HTTP or Firestore triggers

## Interaction Map

- **Auth** → Creates user document in Firestore
- **Chat/AI** → Calls Cloud Functions (Genkit flows), stores messages in Firestore
- **Image Generation** → Calls Cloud Functions, stores images in Cloud Storage
- **Profile** → Reads/writes user document in Firestore
- **Analytics** → Firebase Analytics auto-configured
- **Remote Config** → Feature flags, force update, A/B testing
